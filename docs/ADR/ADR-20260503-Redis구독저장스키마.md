# ADR-20260503 — Redis 구독 저장 스키마

## 상태
승인됨

## 배경
Web Push 구독 정보와 크론 알림 판단에 필요한 위치 정보(측정소명)를 서버에 저장해야 한다. Upstash Redis를 사용하며, Serverless 환경에서 효율적인 자료구조를 선택해야 했다.

## 결정

### Redis 키 구조

```
subscriptions                   Hash
  field: sha256(endpoint)[0:24] # endpoint URL의 SHA-256 앞 24자
  value: JSON string {
    endpoint: string             # Push subscription endpoint URL
    keys: {
      p256dh: string             # 공개키
      auth: string               # 인증 시크릿
    }
    expirationTime: number|null
    stationName: string          # 에어코리아 측정소명 (크론에서 사용)
    regionName: string           # 표시용 지역명
  }

last_pm25:{stationName}         String
  value: "18"                   # 직전 크론 실행 시 PM2.5 수치 (문자열)
```

### Hash 자료구조 선택 이유
- 구독자 전체 목록을 크론에서 한 번에 `HGETALL`로 가져올 수 있음
- 특정 구독자 삭제 시 `HDEL`로 O(1) 처리
- 향후 구독자 수 조회 `HLEN`도 O(1)

### 해시 키 생성
- endpoint URL 전체가 키가 되면 너무 길어 Redis 메모리 낭비
- SHA-256 앞 24자로 단축 → 충돌 확률 무시 가능 수준

## 주의사항
- `last_pm25` 키가 없는 경우(첫 실행) → 이전값 없으므로 알림 미발송 후 값만 저장
- 구독 만료(`expirationTime`) 처리는 크론에서 push 실패 시 해당 구독 삭제로 처리
