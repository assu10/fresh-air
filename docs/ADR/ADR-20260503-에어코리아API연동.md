# ADR-20260503 — 에어코리아 API 연동 방식

## 상태
승인됨

## 배경
브라우저 GPS 좌표(WGS84)를 이용해 가장 가까운 측정소의 PM2.5 데이터를 가져와야 한다. 에어코리아 API는 두 단계의 호출이 필요하고, TM 좌표계를 요구한다.

## 결정

### 사용 API 엔드포인트

| 역할 | 엔드포인트 |
|------|-----------|
| 근접 측정소 조회 | `MsrstnInfoInqireSvc/getNearbyMsrstnList` |
| 실시간 대기질 조회 | `ArpltnInforInqireSvc/getMsrstnAcctoRltmMesureDnsty` |

### 좌표 변환
- 브라우저 GPS는 WGS84(위경도) 반환
- 에어코리아 `getNearbyMsrstnList`는 TM 좌표계(EPSG:5181) 요구
- **외부 라이브러리(proj4) 없이 Snyder TM 공식을 직접 구현** (`lib/geo.ts`)
  - GRS80 타원체, 중부원점 (127°E, 38°N), 거짓 동거 200000m, 거짓 북거 600000m

### API 프록시
- 브라우저에서 에어코리아 API를 직접 호출하면 API 키 노출 위험
- `GET /api/air-quality?lat=&lng=` 서버 사이드 프록시를 통해 API 키를 서버에서만 사용

### 응답 구조
```json
{
  "stationName": "강남구",
  "addr": "서울 강남구",
  "distanceKm": 0.8,
  "pm25": 18,
  "grade": "보통",
  "color": "#22C55E",
  "canVentilate": true,
  "dataTime": "2026-05-03 14:00"
}
```

## 검토한 대안
- **proj4 라이브러리 사용**: 정확하지만 번들 크기 증가, 서버 사이드 전용이라 영향 적음. Snyder 공식 직접 구현으로 의존성 최소화.
- **OpenAQ API**: 글로벌 커버리지이나 한국 데이터 갱신 주기가 에어코리아보다 느림
- **IQAir API**: 무료 플랜 호출 횟수 제한이 있어 30분 폴링에 불리

## 주의사항
- 에어코리아 API는 HTTP(비HTTPS) 엔드포인트 — 서버 사이드에서만 호출 가능
- PM2.5 값이 `"-"` 또는 `null`로 오는 경우(점검중) 처리 필요
- serviceKey는 공공데이터포털에서 이미 URL 인코딩된 형태로 제공되므로 재인코딩 금지
