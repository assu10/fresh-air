# ADR-20260503 — TDD (테스트 주도 개발) 도입

## 상태
승인됨

## 배경
Phase 3까지 구현 코드를 먼저 작성했으나, Phase 4부터 TDD 방식으로 전환하기로 결정됐다.

## 결정
**모든 구현 작업은 TDD(Test-Driven Development) 방식으로 진행한다.**

### TDD 사이클
1. **Red** — 실패하는 테스트 먼저 작성
2. **Green** — 테스트를 통과하는 최소 구현 작성
3. **Refactor** — 코드 정리 (테스트는 계속 통과해야 함)

### 테스트 스택
| 도구 | 용도 |
|------|------|
| Jest 30 | 테스트 러너 |
| @testing-library/react | 컴포넌트 테스트 |
| @testing-library/jest-dom | DOM 매처 확장 |
| @testing-library/user-event | 사용자 인터랙션 시뮬레이션 |
| jest-environment-jsdom | 브라우저 환경 시뮬레이션 |

### 테스트 파일 위치
```
__tests__/
  lib/          # 유틸 함수 단위 테스트
  api/          # API route 통합 테스트
  components/   # 컴포넌트 렌더링/인터랙션 테스트
```

### Phase 완료 기준
해당 Phase의 모든 테스트가 `npm test` 실행 시 통과해야 완료로 간주한다.

## 적용 범위
- Phase 4 (크론 엔드포인트)부터 적용
- Phase 2, 3은 소급 적용하지 않음 (추후 리팩토링 시 테스트 추가 검토)

## 결과
`jest.config.ts`, `jest.setup.ts` 설정 완료.
`package.json`에 `test`, `test:watch`, `test:coverage` 스크립트 추가.
