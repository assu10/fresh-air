# ADR-20260517 — PWA 강화 방식 선택

## 배경

Fresh Air를 Android/iOS에서 앱처럼 사용할 수 있게 하고 싶다.
현재는 웹 브라우저에서만 접근 가능하며, 홈 화면 설치 경험이 없다.

## 검토한 선택지

### 1. Expo (React Native)
- 장점: iOS/Android/Web 단일 코드베이스, Expo Push로 알림 통합 관리
- 단점: React 컴포넌트를 React Native로 재작성 필요, App Store($99/년) + Play Store($25) 비용

### 2. Capacitor
- 장점: 기존 웹 코드 거의 재사용
- 단점: Next.js App Router(SSR)와 호환 문제, 정적 빌드 전환 필요, 스토어 비용 동일

### 3. Flutter
- 장점: 최고 수준의 네이티브 UX
- 단점: Dart 언어 학습, 전면 재작성, 스토어 비용 동일

### 4. PWA 강화 (선택)
- 장점: 기존 코드베이터 유지, 추가 비용 없음, Android/iOS 홈 화면 설치 지원
- 단점: App Store/Play Store 미등록, iOS Web Push는 iOS 16.4+ + standalone 설치 필수

## 결정

**PWA 강화**를 선택한다.

## 근거

- App Store($99/년)·Play Store($25) 비용을 지불하지 않는 방향으로 결정
- 이 앱의 핵심 가치(미세먼지 알림)는 스토어 없이도 PWA로 동일하게 제공 가능
- 기존 Next.js + Vercel 인프라를 그대로 유지할 수 있어 추가 운영 부담 없음
- Android Chrome은 `beforeinstallprompt`로 네이티브에 준하는 설치 경험 제공 가능
- iOS는 Safari "홈 화면에 추가"로 설치 가능, iOS 16.4+ 기준 Web Push도 지원

## 제약 및 트레이드오프

| 항목 | 내용 |
|------|------|
| iOS Web Push | iOS 16.4+ Safari + standalone(홈 화면 설치) 상태 필수 |
| 스토어 미등록 | 검색·추천 노출 없음 — 직접 URL 공유로만 유입 |
| iOS 설치 | 브라우저 내 "홈 화면에 추가" 안내 필요 (자동 프롬프트 불가) |
