# ADR-20260503 — 알림 방식 선택

## 상태
승인됨

## 배경
앱을 닫은 상태에서도 미세먼지 개선 시 즉시 알림을 받아야 한다. 여러 알림 방식 중 선택이 필요했다.

## 결정
**Web Push API (VAPID) + Service Worker** 방식 채택.

- 브라우저에서 푸시 구독 등록 → 서버가 VAPID 키로 서명하여 알림 발송
- Service Worker가 백그라운드에서 push 이벤트를 수신해 알림 표시
- 알림 발송 트리거는 서버 사이드 크론 (cron-job.org → `/api/cron/check`)

## 알림 발송 조건
- PM2.5 **이전값 > 35 μg/m³ AND 현재값 ≤ 35 μg/m³** 전환 시에만 발송
- 이미 좋은 상태가 유지되는 경우 중복 발송 안 함

## 알림 메시지
- 제목: `🌿 지금 환기하기 좋아요!`
- 본문: `PM2.5 {value} μg/m³ — 창문을 열어도 좋습니다.`

## 검토한 대안

- **네이티브 앱 푸시(FCM/APNs)**: 더 안정적이나 iOS/Android 별도 앱 개발 필요, 앱스토어 심사 요구
- **SMS/카카오 알림**: 비용 발생, 번호 등록 필요
- **이메일**: 실시간성 부족
- **클라이언트 폴링**: 앱 열린 상태에서만 동작, 백그라운드 알림 불가

## 주의사항
- Service Worker는 HTTPS 환경에서만 동작 (localhost 예외)
- iOS Safari는 iOS 16.4 이상에서 PWA 푸시 알림 지원
