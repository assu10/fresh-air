## 이슈 1: 알림 토글 동작 안 함

### 원인 A — VAPID 키 미설정

`useNotification`의 `toggle()`이 `pushManager.subscribe()` 호출 시 `applicationServerKey`가 빈 문자열이어서 Web Push 구독이 실패했다.

**수정**: `web-push` CLI로 VAPID 키 페어 생성 후 `.env.local`에 설정.

```
VAPID_PUBLIC_KEY=BBmhgY...
NEXT_PUBLIC_VAPID_PUBLIC_KEY=BBmhgY...  # 브라우저에서 참조하는 키
VAPID_PRIVATE_KEY=QorbeA...
VAPID_SUBJECT=mailto:juhyun10.lee@gmail.com
```

### 원인 B — addr=null 조기 반환

Open-Meteo 폴백 시 `addr`이 `null`이어서 toggle 가드가 조기 반환했다.

```typescript
// 수정 전
if (!registration || !stationName || !addr) return;
// 수정 후
if (!registration || !stationName) return;
```

subscribe API에서 `regionName`은 필수값이 아니므로 `addr` 조건 제거로 충분.

## 이슈 2: 위치 동 단위 표시

### 접근 방법

기존 `addr`은 에어코리아 측정소 주소(e.g. "서울 강남구")였으나, 사용자 실제 위치가 아니다.
GPS 좌표를 역지오코딩해 동 단위 주소를 가져오도록 변경.

**역지오코딩 API 선택**: OpenStreetMap Nominatim (무료, API 키 불필요)

Nominatim 응답에서 한국 동 단위 필드:
- `address.suburb` → 동 (e.g. "역삼1동", "명동") — 가장 일반적
- `address.quarter` → 일부 지역에서 동 대신 사용 (e.g. "태평로1가")
- `address.borough` → 구 (e.g. "강남구") — 최후 폴백

### 구현

**`lib/hooks/useReverseGeocode.ts`** (신규)

```typescript
export function useReverseGeocode(coords: { lat: number; lng: number } | null): {
  locationName: string | null;
  loading: boolean;
}
```

- coords 변경 시 Nominatim 호출 (cleanup으로 race condition 방지)
- 필드 우선순위: `suburb` → `quarter` → `borough`
- 실패 시 locationName=null (에러 상태 표시 없이 graceful 처리)

**`components/LocationDisplay.tsx`**

props 정리: `stationName`, `addr` → `locationName` 단일 prop으로 단순화.

**`app/page.tsx`**

`useReverseGeocode(coords)` 추가, LocationDisplay에 `locationName` 전달.
