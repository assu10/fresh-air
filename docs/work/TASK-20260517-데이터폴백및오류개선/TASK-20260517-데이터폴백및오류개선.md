# TASK-20260517 — 공기질 데이터 폴백 및 오류 개선

## 연관 문서
- [[docs/ADR/ADR-20260517-OpenMeteo폴백도입]]
- [[docs/work/TASK-20260505-PWA프론트엔드/TASK-20260505-PWA프론트엔드]]

## 배경

Phase 5 PWA 프론트엔드 구현 후 로컬에서 실행했을 때 "데이터를 불러올 수 없어요" 오류가 지속 발생했다.
원인 조사 결과 에어코리아 API가 `SERVICE_KEY_IS_NOT_REGISTERED_ERROR`를 HTTP 200으로 반환하고 있었으며,
이를 정상 응답으로 처리하던 것이 문제였다.

## 작업 목표

1. 에어코리아 API 오류 감지 및 적절한 예외 처리
2. API 실패 시 대체 데이터 소스로 폴백
3. 사용자에게 오류 원인 표시 및 재시도 UI 제공
4. 관련 테스트 정비

## 완료된 작업

### 1. 에어코리아 resultCode 검증 (`lib/airkorea.ts`)
에어코리아 API는 오류 시에도 HTTP 200을 반환하며 JSON 본문 `response.header.resultCode`로 성공 여부를 전달한다.
`getNearbyStations`, `getRealtimeAirQuality` 양쪽에 `resultCode !== '00'` 검증을 추가해 오류 시 예외를 던지도록 수정했다.

```typescript
const header = json?.response?.header;
if (header?.resultCode !== '00') {
  throw new Error(header?.resultMsg ?? '에어코리아 API 오류');
}
```

### 2. Open-Meteo 폴백 도입 (`lib/openweather.ts`, `app/api/air-quality/route.ts`)
에어코리아 실패 시 Open-Meteo Air Quality API(무료, API 키 불필요)로 폴백하도록 변경했다.
OpenWeatherMap에서 Open-Meteo로 교체한 이유는 [[docs/ADR/ADR-20260517-OpenMeteo폴백도입]] 참고.

폴백 흐름:
```
1차: 에어코리아 API (한국 공식 측정소 데이터)
  └─ 실패 → 2차: Open-Meteo (위성·모델 기반, 전 세계)
               └─ 실패 → HTTP 500 반환
```

응답에 `source` 필드 추가 (`'airkorea'` | `'openweather'`).
Open-Meteo 폴백 시 `stationName`은 `'현재 위치'`로 표시.

### 3. 다시 불러오기 UI (`components/AirQualityCard.tsx`, `lib/hooks/useAirQuality.ts`)
- `useAirQuality`에 `retry()` 함수 추가 (`retryCount` 상태로 useEffect 재트리거)
- `AirQualityCard`에 `onRetry?: () => void` prop 추가
- 오류 상태에서 "다시 불러오기" 버튼 표시
- 훅이 API 오류 본문(`errJson.error`)을 읽어 실제 오류 메시지를 전달

### 4. 테스트 정비
- `__tests__/openweather.test.ts`: OpenWeatherMap → Open-Meteo 응답 형식으로 업데이트
- `__tests__/api/air-quality/route.test.ts`:
  - `jest.mock()` 호이스팅 문제 수정 (imports 앞으로 이동)
  - `as jest.Mock` → `jest.mocked()` 패턴으로 타입 안전성 개선
