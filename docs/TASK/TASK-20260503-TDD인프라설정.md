# TASK-20260503 — TDD 인프라 설정 (Phase 1.5)

## 연관 ADR
- [[ADR-20260503-TDD도입]]
- [[ADR-20260503-기술스택선택]]

## 작업 목표
Phase 4부터 TDD 방식으로 작업할 수 있도록 Jest 기반 테스트 인프라를 구성하고, 개발 방법론을 문서화한다.

## 완료된 작업

### 의존성 설치
```bash
npm install -D jest jest-environment-jsdom \
  @testing-library/react @testing-library/jest-dom \
  @testing-library/user-event @types/jest
```

### 설정 파일

| 파일 | 내용 |
|------|------|
| `jest.config.ts` | next/jest 기반 설정, jsdom 환경, `@/*` 경로 매핑, 커버리지 수집 대상 |
| `jest.setup.ts` | `@testing-library/jest-dom` 매처 등록 |

### package.json 스크립트 추가
| 명령어 | 역할 |
|--------|------|
| `npm test` | 전체 테스트 실행 |
| `npm run test:watch` | 파일 변경 감지 모드 |
| `npm run test:coverage` | 커버리지 리포트 생성 |

### 문서 업데이트
- `CLAUDE.md` — TDD 방법론 섹션 추가 (사이클, 파일 위치 규칙, Phase 완료 기준)
- `WBS.md` — Phase 1.5 (테스트 인프라) 추가, Phase 4/5에 TDD 작업 항목 반영

## 생성/수정된 파일

|------|------|
| `jest.config.ts` | Jest 설정 |
